# Contributing to AI RACM

Thank you for your interest in contributing to the AI Risk and Control Matrix (AI RACM)! We welcome contributions from audit professionals, risk management experts, compliance specialists, and AI practitioners.

## How to Contribute

### Reporting Issues
- Use GitHub Issues to report bugs, suggest enhancements, or request new control categories
- Provide clear descriptions and examples where possible
- Tag issues appropriately (bug, enhancement, question, etc.)

### Suggesting Improvements
1. **Fork the repository** - Create your own copy of the project
2. **Create a branch** - Use a descriptive name (e.g., `add-gdpr-controls` or `fix-risk-scoring`)
3. **Make your changes** - Update the matrix with your improvements
4. **Test thoroughly** - Ensure your changes work as intended
5. **Submit a Pull Request** - Describe your changes clearly

### Contribution Guidelines

#### What We're Looking For:
- New risk categories and control measures for AI systems
- Industry-specific controls (financial services, healthcare, etc.)
- Regulatory compliance mappings (EU AI Act, GDPR, SOX, etc.)
- Real-world examples and case studies
- Improvements to existing controls and risk assessments
- Documentation enhancements
- Template improvements and usability fixes

#### Quality Standards:
- **Accuracy**: Ensure controls are technically sound and audit-appropriate
- **Clarity**: Use clear, professional language suitable for audit documentation
- **Completeness**: Include risk ratings, control objectives, and testing procedures where applicable
- **Attribution**: Cite sources for regulatory requirements or industry standards

### Code of Conduct
- Be respectful and professional
- Focus on constructive feedback
- Respect differing viewpoints and experiences
- Accept responsibility and learn from mistakes

## Recognition

All contributors will be:
- Listed in the CONTRIBUTORS.md file
- Credited in GitHub's contributor statistics
- Acknowledged in release notes for significant contributions

## Getting Help

If you need help or have questions:
- Open a GitHub Issue with the 'question' label
- Review existing issues and pull requests
- Check the README for project documentation
